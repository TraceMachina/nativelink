Run NativeLink on hardware you control. Suitable for air-gapped environments
and for teams that want full operational ownership.

## When to choose self-host

- You need data residency guarantees that managed Cloud doesn't yet offer.
- You're running on hardware that isn't a public cloud (bare metal, on-prem
  Kubernetes, restricted networks).
- You want to bring your own observability stack and audit logs.
