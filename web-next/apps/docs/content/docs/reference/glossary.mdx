title: Glossary
description: The terms you need to read the rest of the docs.
---

- **Action** — a build step. Inputs (sources, tools), command, expected
  outputs.
- **ActionResult** — what an action produces. Outputs, exit code, stderr,
  timing.
- **CAS** — Content-Addressable Storage. Every artifact stored by hash.
- **Action Cache (AC)** — `Action → ActionResult` lookup.
- **Scheduler** — dispatches actions to workers.
- **Worker** — sandboxed process that runs an action.
- **LRE** — Local Remote Execution. Hermetic builds on your machine.
- **Hermeticity** — the property of a build whose outputs depend only on its
  declared inputs.
