title: Basic configurations
description: Minimal NativeLink configurations for development and small teams.
---

Working configurations for the three most common shapes:

1. **Single-node, in-memory** — perfect for local development. No
   persistence, no external storage.
2. **Single-node, filesystem-backed** — survives restarts. Suitable for a
   small team's shared cache.
3. **Single-node + S3** — the CAS spills to object storage; everything else
   stays local.
