---
title: Contribution guidelines
description: How to land code in NativeLink — what we accept, what we don't, how reviews work.
---

NativeLink is open source. We accept contributions from anyone willing to
follow these guidelines:

- **One change per PR.** Small PRs land. Big PRs get split or rejected.
- **Tests for new behaviour.** Bug fixes need a regression test; features
  need both unit and integration coverage.
- **Commit messages matter.** First line: imperative, ≤72 characters. Body:
  *why*, not *what*.
- **Discuss design first** for any change touching public APIs, scheduler
  policy, or storage format.
