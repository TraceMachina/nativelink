:::warning
Please note that these examples aren't for production use. They're
illustrative and shouldn't be directly copied for production setups.
:::

This section provides examples of deploying NativeLink to an on-premises
Kubernetes setup. The recommended approach is to use the official
[NativeLink Helm chart](https://artifacthub.io/packages/helm/nativelink/nativelink),
which supports a wide range of configurations including cloud object
stores, Redis, ingress, metrics, and autoscaling.

The examples here use a simplified architecture that prioritizes
understandability. You can modify and expand these setups to better suit
your specific needs.

If you have any specific questions about adding NativeLink to existing or
new setups, feel free to ask for help in our
[Slack](https://forms.gle/LtaWSixEC6bYi5xF7).
